## Getting Started

<Accordion type="single" collapsible className="mb-8">
  <AccordionItem value="how">
    <AccordionTrigger>How does Convex UI work?</AccordionTrigger>
    <AccordionContent>
      Convex UI uses the **shadcn/ui registry system** to install components directly into your project:

      <Command name="password-based-auth-nextjs" />

      This single command:
      - Downloads React components to your `src/components/` directory
      - Adds Convex backend functions to your `convex/` directory
      - Installs any required npm dependencies
      - Creates environment variable templates

      <Callout type="info" className="mt-4">
        Unlike traditional npm packages, these files become **your code**. Customize anything—no upstream updates to worry about.
      </Callout>
    </AccordionContent>

  </AccordionItem>

  <AccordionItem value="overwrite">
    <AccordionTrigger>Will it overwrite my existing files?</AccordionTrigger>
    <AccordionContent>
      **No.** The CLI is safe to run in existing projects:

      | Scenario | Behavior |
      |----------|----------|
      | New file | Created automatically |
      | Existing file (same content) | Skipped |
      | Existing file (different content) | You're prompted to choose |

      <Callout type="warn" className="mt-4">
        **Pro tip:** Always commit your changes to Git before adding components. This gives you a clean diff to review.
      </Callout>
    </AccordionContent>

  </AccordionItem>

  <AccordionItem value="existing-project">
    <AccordionTrigger>Can I use this in an existing project?</AccordionTrigger>
    <AccordionContent>
      Yes! Convex UI works great with existing projects. The components are designed to integrate alongside your existing code.

      **Before running commands:**
      1. Commit any pending changes to Git
      2. Review what files will be added
      3. Run the add command
      4. Check the Git diff to see exactly what changed

      If you already have Convex or shadcn/ui set up, the components will use your existing configuration.
    </AccordionContent>

  </AccordionItem>

  <AccordionItem value="quickstart-needed">
    <AccordionTrigger>Do I need to set up Convex first?</AccordionTrigger>
    <AccordionContent>
      Yes, but it's fast. Convex setup is just two commands:

      <CodeBlock language="bash">

{`npm install convex @convex-dev/auth
npx convex dev`}
</CodeBlock>

      That's it. No dashboard configuration, no account creation, no complex setup. Just run `convex dev` and start building.

      <Callout className="mt-4">
        See the [Quick Start](/ui/docs/getting-started/quickstart) guide for full setup instructions.
      </Callout>
    </AccordionContent>

  </AccordionItem>
</Accordion>

## Technical Questions

<Accordion type="single" collapsible className="mb-8">
  <AccordionItem value="realtime">
    <AccordionTrigger>How does real-time work?</AccordionTrigger>
    <AccordionContent>
      Convex provides **automatic real-time subscriptions**. When data changes anywhere, your UI updates instantly.

      <CodeBlock language="tsx">

{`// This automatically re-renders when messages change
const messages = useQuery(api.messages.list, { roomId: "general" });`}
</CodeBlock>

      **No extra setup required:**
      - No WebSocket configuration
      - No polling intervals
      - No cache invalidation
      - No optimistic update boilerplate

      This powers features like real-time chat, presence indicators, and collaborative cursors in Convex UI.
    </AccordionContent>

  </AccordionItem>

  <AccordionItem value="react-only">
    <AccordionTrigger>Why React only?</AccordionTrigger>
    <AccordionContent>
      Three reasons:

      1. **shadcn/ui is React-based** — Convex UI builds on top of it
      2. **Convex has first-class React support** — Hooks like `useQuery` and `useMutation` make real-time seamless
      3. **It's not a new UI library** — It's established tools (shadcn/ui + Convex) working together

      <Callout type="info" className="mt-4">
        The backend functions in `convex/` work with any frontend. Only the React components are framework-specific.
      </Callout>
    </AccordionContent>

## Troubleshooting

<Accordion type="single" collapsible>
  <AccordionItem value="env-vars">
    <AccordionTrigger>My components aren't connecting to Convex</AccordionTrigger>
    <AccordionContent>
      Check your environment variables:

      <CodeBlock language="env">

{`# Required in .env.local
CONVEX_DEPLOYMENT=dev:your-project-name
NEXT_PUBLIC_CONVEX_URL=https://your-project.convex.cloud`}
</CodeBlock>

      For auth components, you also need:

      <CodeBlock language="env">

{`CONVEX_SITE_URL=http://localhost:3000`}
</CodeBlock>

      <Callout type="info" className="mt-4">
        Run `npx convex dev` to see your actual values in the terminal output.
      </Callout>
    </AccordionContent>

  </AccordionItem>

  <AccordionItem value="schema-conflict">
    <AccordionTrigger>Schema type errors after adding components</AccordionTrigger>
    <AccordionContent>
      If you see TypeScript errors in `convex/schema.ts`, you likely have conflicting table definitions.

      **To fix:**
      1. Open `convex/schema.ts`
      2. Merge any duplicate table definitions
      3. Ensure all index names are unique
      4. Run `npx convex dev` to validate

      <Callout type="warn" className="mt-4">
        Each table can only be defined once. Combine fields from both definitions if needed.
      </Callout>
    </AccordionContent>

  </AccordionItem>

  <AccordionItem value="auth-not-working">
    <AccordionTrigger>Authentication isn't working</AccordionTrigger>
    <AccordionContent>
      Common auth issues and fixes:

      | Issue | Solution |
      |-------|----------|
      | "Invalid URL" error | Check `CONVEX_SITE_URL` matches your dev server |
      | OAuth redirect fails | Verify callback URLs in provider settings |
      | Session not persisting | Ensure `ConvexAuthNextjsServerProvider` wraps your app |
      | "Not authenticated" | Check that `proxy.ts` (Next.js) is in `src/` |

      See the [auth troubleshooting guide](/ui/docs/nextjs/password-based-auth#troubleshooting) for more details.
    </AccordionContent>

  </AccordionItem>
</Accordion>
